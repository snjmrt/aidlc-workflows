# 👋 AI-DLC（AI-Driven Development Life Cycle）へようこそ！ 👋

あなたのニーズに合わせて自動的に適応するソフトウェア開発ワークフローを案内します。

## AI-DLC とは？

AI-DLC は、プロジェクトのニーズに合わせて適応する、構造化されつつ柔軟なソフトウェア開発プロセスです。経験豊富なソフトウェアアーキテクトがいると考えてください。以下を行います:

- **要件を分析** し、必要に応じて確認質問を行う
- **複雑さとリスクに基づいて最適なアプローチを計画** する
- **単純な変更では不要なステップを省略** し、複雑なプロジェクトには包括的な対応を提供
- **すべてを文書化** して、意思決定と根拠の完全な記録を残す
- **明確なチェックポイントと承認ゲートで各フェーズを案内** する

### フェーズ内訳:

**INCEPTION フェーズ** - *計画 & アプリケーション設計*
- **目的**: 何を作るか（WHAT）となぜ作るか（WHY）を決定
- **活動**: 要件理解、既存コードの分析（ある場合）、アプローチの計画
- **成果物**: 明確な要件、実行計画、並列開発のための作業単位数の決定
- **あなたの役割**: 質問に回答、計画のレビュー、方向性の承認

**CONSTRUCTION フェーズ** - *詳細設計、実装 & テスト*
- **目的**: どのように作るか（HOW）を決定
- **活動**: 詳細設計（必要時）、コード生成、包括的なテスト
- **成果物**: 動作するコード、テスト、ビルド手順
- **あなたの役割**: 設計レビュー、実装計画の承認、結果の検証

**OPERATIONS フェーズ** - *デプロイ & 監視（将来）*
- **目的**: どのようにデプロイし運用するか
- **状態**: 将来のデプロイと監視ワークフローのためのプレースホルダー
- **現状**: ビルドとテスト活動は CONSTRUCTION フェーズで扱う

## 主要原則:

- ⚡ **完全適応型**: 各ステージはニーズに応じて独立に評価
- 🎯 **効率的**: 単純な変更は必要最小限のステージのみ実行
- 📋 **包括的**: 複雑な変更はあらゆる安全策を含めて対応
- 🔍 **透明性**: 作業開始前に実行計画を確認・承認できる
- 📝 **記録**: すべての意思決定と変更の完全な監査ログを残す
- 🎛️ **ユーザー主導**: ステージの追加/除外を要求できる

## 次に起こること:

1. **ワークスペースを分析** して新規/既存プロジェクトかを判断
2. **要件を収集** し、必要に応じて確認質問
3. **実行計画を作成** し、提案ステージと理由を提示
4. **あなたが計画をレビューして承認**（または変更依頼）
5. **主要ステージごとにチェックポイントを設けて実行**
6. **完全なドキュメントとテスト付きの動作コード** を提供

AI-DLC プロセスは以下に適応します:
- 📋 あなたの意図の明確さと複雑さ
- 🔍 既存コードベースの状態
- 🎯 変更の範囲と影響
- ⚡ リスクと品質要件

それでは始めましょう！
